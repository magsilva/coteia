Os dados provenientes do usuário são submetidos através do método POST (passo 1) ao processo CriaXML, que realiza o seguinte processamento:
  • integração com outras informações: dados dos usuários (representado pelo passo 3), como login ou ip da máquina; e informações do ambiente (representado pelo passo 4), como data de criação do documento ou título da swiki;
  • padronização de caixa: tags inseridas pelo usuário no formulário HTML são sempre convertidas em caixa baixa para ficarem em conformidade com o DTD;
  • conversão de entidades HTML: entidades, como caracteres de acentuação e símbolos especiais, são convertidas em entidades HTML e, posteriormente, validades pela gramática definida pelo DTD;
  • criação de ligações: os usuários da CoWeb-ICMC podem definir ligações dentro da ferramenta atrave`s de diretrizes próprias, como <link>, inseridas no formulário. Conseqüentemente, realiza-se o seguinte processamento:
    • Extração do conteúdo X da tag <link>
    • Busca ao identificador Y da swiki através da consulta à base de dados (passo 5)
    • Comparação de X com os títulos das páginas cadastradas em Y
    • Se X já foi cadastrado:
      • Retorno do índice que aponta para o hiperdocumento indicado pelo link.
      • Geração da âncora HTML correspondente ao hiperdocumento.
    • Senão:
      • Retorno de um novo índice para o documento cadastrado.
      • Direcionamento do link para a interface de criação de hiperdocumentos.
Como resultado do processo CriaXML, as informações são armazenadas na base de dados da aplicação e um arquivo físico XML é criado (passo 6). Ao final do processo CriaXML, é ativado (passo 7) o processo GeraXHTML, detalhado na figura, que tem como sub-processos:
  • Validação do documento: associa o documento XML a uma gramática (DTD) para validá-lo (passo 8).
  • Associação de folha de estilo (passo 9): especifica e associa uma folha de estilo XSLT ao documento XML validado para o tratamento da apresentação.
  • Processamento: envia os arquivos XML validados e folha de estilo XSLT ao processador Java XT, obtendo como saída (passo 10) um arquivo físico XHTML.
Este processamento é realizado somente a cada edição ou criação de novos hiperdocumentos, permitindo um bom desempenho.
